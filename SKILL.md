# GEO Toolkit — 生成式引擎优化

GEO（Generative Engine Optimization）是优化内容以被 AI 引擎引用的方法论。
目标平台：**Perplexity** / **ChatGPT Search** / **Google AI Overviews** / **Bing Copilot** / **Claude**

## 与传统 SEO 的核心区别

| 维度 | 传统 SEO | GEO |
|------|---------|-----|
| 目标 | 在 SERP 排名前10 | 被 AI 生成答案引用/提及 |
| 信号 | 关键词、外链、技术指标 | 可信度、结构化、权威引用 |
| 结果形式 | 蓝色链接列表 | AI 综合回答中的来源标注 |
| 流量模式 | 点击进入网站 | 品牌曝光 + 部分点击来源 |
| 核心内容要求 | 关键词密度 + 长度 | 直接答案 + 可验证事实 + 权威信号 |

## 路由规则

| 用户输入 | 执行模块 |
|---------|---------|
| 只提供 URL | M1 AI可见性 → M2 可引用性分析 |
| 提供品牌词/行业词 | M1 AI可见性 → M3 竞品AI对比 |
| "内容优化" + URL/文章 | M2 → M4 GEO内容改写 |
| "实体优化" | M5 实体建设 |
| "完整GEO分析" | M1 → M2 → M3 → M4 → M5 |
| 输入不明确 | 询问：提供URL还是品牌词？想分析哪个AI平台？ |

每个模块结束后询问：继续下一模块还是深挖当前结果？

## M1：AI 可见性审计

**目标：** 量化品牌/产品在各 AI 引擎中的出现率和质量

**执行步骤：**

1. 根据品牌/URL，构建三类测试查询：
   - **品牌词**：`"[品牌名]" 是什么`、`[品牌名] review`、`[品牌名] vs [竞品]`
   - **行业词**：`最好的 [产品类别]`、`[行业] 推荐工具`、`[行业] 解决方案`
   - **问题词**：`如何 [用户痛点]`、`[产品功能] 怎么做`

2. 用 WebSearch 搜索各平台上的查询结果（降级方案），同时：
   - 用 web_access 打开 [Perplexity](https://www.perplexity.ai/)，测试上述查询
   - 用 web_access 打开 [ChatGPT](https://chatgpt.com/)，测试上述查询

3. 记录每次测试：
   - 是否被提及（✓/✗）
   - 提及位置（第几段、是否首位）
   - 引用来源（哪个页面被引用）
   - 描述质量（正面/中性/负面）

4. 计算 **AI 可见性得分**（被提及次数 / 总测试次数）

**输出格式：**
```markdown
## AI 可见性报告：{品牌}

### 可见性得分：{X}/{总测试数}（{%}）

| 平台 | 查询类型 | 被提及 | 引用页面 | 描述质量 |
|------|---------|-------|---------|---------|
| Perplexity | 品牌词 | ✓ | /about-us | 正面 |
| ChatGPT | 行业词 | ✗ | - | - |

### 关键发现
- 哪些查询场景有曝光，哪些完全缺失
- 被引用的是哪类内容页面
- 竞品在同类查询中的表现对比
```

## M2：内容可引用性分析

**目标：** 诊断现有内容是否具备被 AI 引用的结构和信号

**执行步骤：**

1. WebFetch 目标 URL，分析页面内容
2. 按 AI 引用信号检查表逐项评分（详见 `references/citation-signals.md`）
3. WebFetch 已被 AI 引用的竞品页面，对比结构差距

**六大引用信号维度：**

**① 直接答案层（Answer Layer）**
- 是否有 ≤ 50 字的直接答案段落（紧跟 H1/H2 后）
- 是否定义了核心概念（"X 是指..."格式）
- FAQ 区块是否存在

**② 数据可信度**
- 是否引用了第三方数据源（研究报告/权威机构）
- 统计数据是否标注年份和来源
- 是否有可验证的具体数字（百分比/金额/数量）

**③ 内容结构**
- 是否有清晰的 H2/H3 层级（AI 用于定位段落）
- 是否有表格、对比图、有序列表
- 段落是否简短聚焦（≤ 4 句/段）

**④ E-E-A-T 信号**
- 作者是否有可识别的专业背景
- 是否显示发布/更新日期
- 是否引用了行业权威来源
- 外链数量和质量

**⑤ Schema 标记**
- FAQPage schema
- HowTo schema
- Article/BlogPosting schema（含 author、datePublished）
- Organization schema

**⑥ 实体明确性**
- 品牌全称是否在页面内完整出现
- 是否有明确的产品/服务定义
- 是否链接到权威第三方来源（Wikipedia、行业报告）

**输出格式：**
```markdown
## 内容可引用性分析：{URL}

### 综合评分：{X}/100

| 维度 | 得分 | 问题 | 修复建议 |
|------|------|------|---------|
| 直接答案层 | 6/20 | 无定义段落 | 在 H1 后加 ≤50字答案 |
| 数据可信度 | 8/20 | 无来源引用 | 引用3条权威数据 |
| 内容结构 | 14/20 | 段落过长 | 拆段至≤4句 |
| E-E-A-T | 10/20 | 无作者信息 | 添加作者Bio |
| Schema标记 | 0/10 | 无任何Schema | 添加FAQ+Article Schema |
| 实体明确性 | 6/10 | 品牌定义模糊 | 首段明确品牌定义 |

### P0 立即修复
### P1 优先优化
```

## M3：竞品 AI 存在感分析

**目标：** 找出竞品被 AI 引用的内容特征，发现可复制的策略

**执行步骤：**

1. 确定竞品列表（用户提供或自动识别）
2. 对竞品执行 M1 相同的查询测试
3. WebFetch 竞品被引用的具体页面，分析：
   - 内容格式（FAQ/定义/数据/教程）
   - Schema 使用情况
   - 页面权威信号
4. WebSearch：`site:{competitor} FAQ OR "what is" OR statistics`
5. 对比可见性得分差距

**输出格式：**
```markdown
## 竞品 AI 存在感对比

| 竞品 | AI可见性得分 | 被引用最多的内容类型 | 核心优势 |
|------|------------|------------------|---------|
| 竞品A | 7/10 | FAQ页面 + 定义类文章 | Schema完整，答案直接 |
| 竞品B | 4/10 | 数据报告 | 权威数据被频繁引用 |
| 你的品牌 | 1/10 | 无 | - |

### 可复制的内容策略
- [策略] - [竞品实例] - [为什么被AI引用]
```

## M4：内容 GEO 改写

**目标：** 将现有内容改写为 AI 友好格式，提升引用概率

**执行步骤：**

1. 读取目标内容（URL 或用户粘贴）
2. 参考 `references/content-patterns.md` 的改写模板
3. 按优先级输出改写建议或完整改写版本：

**改写五步法：**

**Step 1 - 加 Answer Layer**
在每个 H2 标题后立即加入 ≤50字直接答案：
```markdown
## 什么是私人飞机空腿机票

空腿机票是私人飞机从上一个目的地飞回基地时的空载航段，通常以正常价格的30-75%出售。
```

**Step 2 - 数据锚定**
每 500 字至少引用 2 条可验证数据，格式：
`"根据 [权威来源] [年份] 数据，[具体数字]..."`

**Step 3 - FAQ 块**
在文章末部添加 FAQ 区块（5-8条），直接回答用户常见问题：
```markdown
## 常见问题

**Q: 空腿机票安全吗？**
A: 是的。空腿机票使用与正常包机相同的飞机和机组人员...

**Q: 如何找到空腿机票？**
A: 可通过专业平台（如 JETBAY）设置路线提醒...
```

**Step 4 - Schema 模板**
输出对应的 JSON-LD 代码（FAQPage / HowTo / Article），详见 `references/content-patterns.md`

**Step 5 - 内部实体链接**
确保页面明确定义品牌，并链接到权威来源

## M5：实体建设

**目标：** 确保品牌作为独立实体存在于 AI 的知识库中

**AI 如何识别实体：** AI 引擎通过训练数据认识品牌实体，主要来源：
- Wikipedia / Wikidata 词条
- Google 知识面板（Knowledge Graph）
- 权威媒体报道（TechCrunch、Forbes、行业垂媒）
- LinkedIn Company Page
- Crunchbase 公司档案
- 官方网站的 Organization Schema

**执行步骤：**

1. WebSearch：`"{品牌名}" wikipedia OR wikidata`
2. WebSearch：`"{品牌名}" site:crunchbase.com OR site:linkedin.com`
3. WebSearch：`"{品牌名}" site:techcrunch.com OR site:forbes.com`
4. 检查官网 Organization Schema（WebFetch 抓取验证）

**实体建设清单：**

| 实体信号 | 检查 | 优先级 |
|---------|------|-------|
| Wikipedia 词条 | 是否存在 | P2（有门槛，需达到"可关注度"） |
| Wikidata 词条 | 是否存在 | P1（可自行创建） |
| Google 知识面板 | 是否触发 | P1 |
| Crunchbase 档案 | 是否完整 | P1 |
| LinkedIn 公司页 | 是否完整 | P0 |
| Organization Schema | 是否部署 | P0 |
| 权威媒体提及 ≥3 | 数量是否达标 | P1 |

**输出：** 实体建设行动清单（按优先级排序）
